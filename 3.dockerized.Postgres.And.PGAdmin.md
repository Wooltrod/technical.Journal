# Setting up a Postgres Server, with PGAdmin interface, entirely through Docker

- Create a docker-compose, YAML file (YAML Ain't Markup Language) - it contains important parameters for the Docker images, Port Mapping for access on the Host Machine, and grouping the containers into a common "network"

**docker-compose.yaml**

```CLI
services:
  db:
    image: postgres
    container_name: PG_container
    restart: always
    environment:
      POSTGRES_USER: myUsername
      POSTGRES_PASSWORD: abcdefghijklmnop
      POSTGRES_DB: myUsername_db
    ports:
      - "5432:5432"
    volumes:
      - pgdata:/var/lib/postgresql
    networks:
      - pg-network

  pgadmin:
    image: dpage/pgadmin4
    container_name: PGAdmin_container
    restart: always
    environment:
      PGADMIN_DEFAULT_EMAIL: myUsername@gmail.com
      PGADMIN_DEFAULT_PASSWORD: ghijklmnopqrstuvw
    ports:
      - "80:80"
    volumes:
      - pgadmin-data:/var/lib/pgadmin
    networks:
      - pg-network
    depends_on:
      - db

volumes:
  pgdata:
  pgadmin-data: 

networks:
  pg-network:
    driver: bridge
```

- Save the docker-compose.yaml file to a folder of your choice. To run the containers, use a Linux-based terminal like GitBash, anywhere on your Windows PC (you must have Windows Subsystem for Linux installed) - a good option would be to use Docker Desktop's built-in terminal. Now, navigate to the directory where you saved the docker-compose.yaml file, and issue the following command:

```CLI
 docker compose up -d
```

- Verify that both containers are running by issuing the following commands:

```CLI
docker ps
```

- or;

```CLI
docker ps -a
```

- To obtain the IP Address of the running Postgres container (the output can be used in the Host name/address field instead of the service name 'db' when logging into our Postgres instance):

```CLI
docker inspect -f '{{range .NetworkSettings.Networks}}{{.IPAddress}}{{end}}' PG_container
```

- Wait for around 5 minutes for PGAdmin, after which you can access it from the browser on "localhost:80" (or whichever port you mapped).

- For login, provide exact credentials used for configuring the PGAdmin in your docker-compose.yaml file - email and password.

## Adding a Server ([Left Tab] Servers -> Register -> Server):

**Below is how the fields of the dialog match to what we have configured on the docker-compose.yaml file:**

    **General Tab**

    - Name <-> Enter a custom name for your Server (may contain several databases)

    **Conncetions Tab**

    - Host name/address <-> "db" (or the running Postgres container's IP Address)
    - Port <-> 5432 (default)
    - Maintenance databse <-> postgres (default)
    - Username <-> POSTGRES_USER
    - Password <-> POSTGRES_PASSWORD

- Click the "Save" button.

## Creating a table in conjunction with data from csv file:

- Access the query tool for your database, by right clicking on the database with the name specified next to "POSTGRES_DB" in you docker-compose.yaml file (Servers -> server_name -> Databases -> myUsername_db(right-click)).

- In line with the columns of your CSV table, and for smooth importing of the csv data, prepare/create a table using the CREATE TABLE sql command as follows:

```CLI
CREATE TABLE public.table_name (
    id SERIAL,          
    field_1 VARCHAR(100), 
    field_2 VARCHAR(100)    
);
```

- Confirm that the named, empty table has been created by navigating as follows:
    **Servers -> server_name -> Databases -> myUsername_db -> Schemas -> public -> Tables**

- Right-click on your table and select the "Import/Export data" option to import the csv file. In the pop-up dialog, select the appropriate format as csv. Next, Move to the "Options" tab and toggle "Header" on depending on whether your data set has a top row of headers.
