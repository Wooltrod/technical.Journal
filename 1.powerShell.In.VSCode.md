# Moving from Git Bash to VS Code's PowerShell Terminal

For a long time, I ignored the potential of Microsoftâ€™s native PowerShell, opting instead for Git Bash to run Python scripts for my personal projects.

Recently, I decided to explore working directly with PowerShell inside VS Code. Here are some practical notes on directory navigation and running Python scripts.

---

## Directory Navigation & Management

### 1. Move to the Root Directory of Your Project Drive

In my case, my project folder is located in the `D:` partition. To switch drives in PowerShell:

```powershell
PS C:\Windows\System32> cd d:\
```

This switches the session from `C:` to `D:`.

---

### 2. List Folders in a Drive

To view the structure of the current drive (for example, `D:\`) and confirm your project folder exists:

```powershell
PS D:\> dir
```

This lists all directories and files in the current location.

---

### 3. Navigate to a Specific Project Folder

- Type `cd ` (with a space)
- Press `Tab` repeatedly to cycle through folder names until your desired folder is displayed

Example:

```powershell
PS D:\> cd <Tab>
```

With each <Tab> keypress, PowerShell will display a director name within the folder.

---

### 4. Bonus Navigation Tips

#### a) Move Up One Directory Level

```powershell
PS D:\My Project> cd ..
```

#### b) Move to Your User Home Directory

```powershell
PS D:\> cd ~
```

---

## Running Python Scripts in PowerShell

You can call Python using its full executable path (Assuming Python is not yet installed and added to your system PATH), and run a Python script from within your project directory like this:

```powershell
PS D:\My Project> & "C:\Path\To\Python\python.exe" script.py
```

Notes:

- The `&` is the call operator in PowerShell.
- Quotes are required if the path contains spaces.
- `script.py` must be in the current directory (or you must provide its full path).
